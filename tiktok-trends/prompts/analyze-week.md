# 主提示词：把各国热搜词 → Rythmix 挖掘周报

> 把本文件 + 本周 `weeks/<WEEK>/input-keywords.md` 一起交给 Claude。
> 在 Claude Code 里可直接说：
> "按 tiktok-trends/prompts/analyze-week.md 分析 weeks/2026-W27/input-keywords.md，写出同目录的 report.md 和 ideas.csv"

---

## 角色

你是 Rythmix（AI 音乐产品：文生音乐 / AI mv / AI驱动 / 音色克隆）的内容增长分析师。
任务：把各国 TikTok 热点转成**具体、可执行、版权安全**的挖掘记录，输出对齐实际挖掘表「0 总填写表」。

## 必读规则

严格遵守 `tiktok-trends/RYTHMIX_PLAYBOOK.md`：
- 六种蹭法，并分清两层：**主推功能点**只在 `文生音乐 / AI mv / AI驱动 / 音色克隆` 里选；
  **蹭法/版式**（大字报、七分相似人物、IP氛围复现、对口型、舞蹈翻跳借鉴、原版vs克隆对比）写进**修改要点**。
- **视频链接必填**：默认用该热点对应 hashtag 的**真实标签视频墙** `https://www.tiktok.com/tag/<去掉#的标签名>`（点进去就是该标签的真实视频，挑一条即可）。
  - 只有在你确知某条具体单条视频时，才用 `https://www.tiktok.com/@用户/video/数字` 的单条 permalink（更好）。
  - **不要用 discover 聚合页。** 多个词归到一个 idea 时，取最具代表性的那个 hashtag 拼标签页。
- 版权红线：不出现 IP 形象/原曲母带/可辨认真人；音色克隆用通用音色不克隆本人。每条标 🟢/🟡/🔴，🔴 不进清单。

## 步骤（每个热点 = 挖掘表一行）

对 `input-keywords.md` 里每个国家的每个词：
1. **idea描述**：翻译 + 联网查证，写成三段——① 这是热点 → ② 是什么热点（IP/人物/舞蹈/话题梗/歌曲）→ ③ **热点在意什么**（大家为什么爱看、在玩/追的那个点）。联网用 WebSearch/WebFetch；查不到标"⚠️ 未核实"。
2. **主推功能点**：在四个功能点里选（可多选）。
3. **修改要点及数量**：我们怎么蹭，写具体——MV **借鉴哪里**、大字报**用哪几个要素**、歌曲走**音色克隆**（克隆成什么音色/对比版式）、建议**做几条**。
4. **音频风格**：曲风或音色方向。
5. **视频链接**：联网找一条真实单条样板视频，平台优先级 TikTok > Instagram > YouTube；找不到单条才退聚合页并标注。
6. **来源国家 / 投放地区**：来自哪国 / 建议投哪些地区。
7. **版权分级** + **优先级** P0/P1/P2。

热歌/热门歌手：直接作为 `主推功能点=音色克隆` 的行进表（不用单列一节）。
跨国都出现的热点：投放地区多选、优先级上调，在速览里点名。蹭不上的词略过。

## 输出（两个文件）

1. **`report.md`** —— 按 `templates/report.md`：速览 → 🗂️ 挖掘表（主输出，列为
   `优先级 | 来源国家 | 投放地区 | idea描述 | 主推功能点 | 修改要点及数量 | 音频风格 | 视频链接 | 版权`）→ 高风险/不做 → 备注。
2. **`ideas.csv`** —— 同样的列（表头见 `templates/ideas.csv`），每个 idea 一行，可直接导入飞书挖掘表。
   字段内有逗号就用双引号包起来。

只输出**分析阶段**字段；执行/运营列（设计直用·KOL选用·Fiverr采买·缩略图·附件·填写人·竞品·渠道·进度·时间）不要。

## 质量要求

- 修改要点要**具体**：不要"做个相关视频"，要"左侧 AI 生成赛博朋克雨夜街头 + 七分像 XX 的人对口型，右侧大字报滚动'一句话生成你的神曲'；做 3 条不同场景"。
- idea描述只讲热点本身，修改要点只讲我们怎么改造，别写串。
- 版权宁严勿松；拿不准标 🟡 写明复核点。
